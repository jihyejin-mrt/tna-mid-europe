이슈 드리븐 분석을 수행한다. 아래 논점이 주어진다:

<논점>
$ARGUMENTS
</논점>

**출력 규칙:**
- 두괄식: 8단계 시작 전, "결론: [핵심 한 문장]"으로 시작한다
- 중요한 수치는 **굵은 글씨**, 테이블·시트명은 `코드 포맷`으로 표시한다
- So What? / Why So? 검증을 모든 주장에 적용한다
- 논점이 불명확하면 먼저 물어본다

CLAUDE.md의 출력 포맷 (8단계)을 엄격하게 따른다:

1. **논점 정의** — 본질적 문제를 한 문장으로
2. **초기 가설** — Day One Answer + 근거
3. **조사 결과와 사실 (空)** — 검증에 필요한 데이터·정보
4. **해석 (雨)** — 사실들의 의미
5. **선택지 (MECE)** — 표 형식 (옵션 / 개요 / 메리트 / 디메리트)
6. **추천 시책 (傘)** — 하나로 좁혀 이유 명시. "두 안 모두 유효" 금지
7. **실행 로드맵** — 누가 / 무엇을 / 언제까지 / 어떻게
8. **예상 리스크와 대책** — 과제별 대응책
